git

Git is a Distributed Version Control System.

What is that and why do you need one?

Git is not just github,
Github is one service provider among others.

What is inside

A git repository is a database that stores a directed graph where each node is a version of the project, containing the file sysem as it looks in that version.

Each node is identified by a hash.

There can be symbolic names (tags and branches) pointing to some versions.

Tree / repo / stage

You have a directory containing one checked-out version of your project.

You have a repository containing the graph of versions of your project.

You have a staging area for your next commit.

Some commands

git init dirname
cd dirname
git status
git add filename(s)
git commit -m message
git checkout version filename(s)
git checkout -b branchname

Workig with others

Github, gitlab, ssh, ...

git clone repo-url
git remote add alias another-repo-url
git fetch alias
git push alias

Pull requests

Continous integration

A bot can check out each commit as it is published, and perform automatic testing and even deploy code to production.

Hooks

You can cofigure git to do automatic tasks each time you do something (e.g. run a code formatter before each checkin).
